DllInstall

UNDOCUMENTED WIN 7 / NT 6.1 ntvdm64.dll

Description: Installation function for NTVDM registry entries. This function is called by `regsvr32` or during system installation to populate the registry with information about 16-bit applications that need special handling. It iterates through a hardcoded table of application entries (starting at 0x10001388) and registers each one under `HKLM\Software\Microsoft\Windows NT\CurrentVersion\NtVdm64`. Each entry contains the command line, product name, internal name, and product version for the 16-bit application.

C# P/Invoke Structure:

[DllImport("ntvdm64.dll", CharSet = CharSet.Unicode)]
public static extern int DllInstall(
    bool bInstall,              // TRUE = install, FALSE = uninstall
    string pszCmdLine           // Command line parameters (optional)
);

Installation Table Structure (0x10001388):

[StructLayout(LayoutKind.Sequential, CharSet = CharSet.Unicode)]
public struct VDM_APP_ENTRY {
    public string ProductName;      // +0x00 - Application display name
    public string ProductVersion;   // +0x04 - Version string
    public string InternalName;     // +0x08 - Internal executable name
    public string CommandLine;      // +0x0C - Command line to execute
    public string Unknown;          // +0x10 - Possibly MappedExeName
    public string Unknown2;         // +0x14 - Possibly additional parameters
}

// Known entries in the table (from decompilation):
// Entry 1: INSTALLSHIELD5 related
// Entry 2-8: Various InstallShield versions
// Total entries: 8 (0xA8 bytes / 0x18 bytes per entry)

Registry Keys Created:

HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\NtVdm64
├── [Application Subkey]
│   ├── CommandLine (REG_SZ)     - Command line to execute
│   ├── ProductName (REG_SZ)     - Display name
│   ├── InternalName (REG_SZ)    - Internal executable name
│   ├── ProductVersion (REG_SZ)  - Version string
│   └── MappedExeName (REG_EXPAND_SZ) - Expanded executable path
└── ... (multiple subkeys for different applications)

Example entry for INSTALLSHIELD5:
HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\NtVdm64\INSTALLSHIELD5
    CommandLine = "C:\Windows\SysWOW64\msiexec.exe /i setup.msi"
    ProductName = "InstallShield Setup"
    InternalName = "ISSetup.exe"
    ProductVersion = "5.0.0.0"
    MappedExeName = "%SystemRoot%\SysWOW64\msiexec.exe"

Return Codes (HRESULT):

0x00000000 - S_OK - Installation completed successfully
0x80004005 - E_FAIL - Registry operation failed
    [0x00000000]
    [0x80004005]

⚠️ WARNING:

This function requires administrator privileges to write to HKLM. It is typically called by `regsvr32 ntvdm64.dll` or during Windows installation. Modifying these registry entries can affect how 16-bit applications are launched. Incorrect entries can cause 16-bit applications to fail to start or run with wrong parameters. This function is not intended for direct use by applications.
