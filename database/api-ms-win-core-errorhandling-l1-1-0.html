api-ms-win-core-errorhandling-l1-1-0.dll

API SET FORWARDER KERNEL32.DLL Windows 7+ (with KB2999226) 7 exports

Description: This is an API Set forwarder DLL for error handling and exception management in Windows. It contains NO executable code — only export forwarders to kernel32.dll. These functions provide the foundation for Windows error reporting, structured exception handling (SEH), and last-error management used by virtually every Windows application.

Why it exists: This API Set represents the core error handling infrastructure of Windows. GetLastError/SetLastError are called by almost every Win32 application. RaiseException is the foundation of Structured Exception Handling (SEH) — the mechanism that powers __try/__except and __try/__finally in C/C++. UnhandledExceptionFilter is what shows the "Program has stopped working" dialog.

📋 Complete Function List (7 exports)

Function Description Key Feature
GetErrorMode Retrieves the error mode for the current process Controls system error dialog behavior
GetLastError Returns the last-error code for the calling thread Most called Win32 function!
RaiseException Raises an exception in the calling thread (SEH) Foundation of C++ exceptions and structured exceptions
SetErrorMode Controls whether the system handles errors or the application SEM_FAILCRITICALERRORS, SEM_NOGPFAULTERRORBOX
SetLastError Sets the last-error code for the calling thread Used by APIs to report failure reasons
SetUnhandledExceptionFilter Installs a top-level exception handler Custom crash reporting (crashpad, breakpad, etc.)
UnhandledExceptionFilter The default unhandled exception filter Shows the "Program has stopped working" dialog

📊 Error Mode Flags (SetErrorMode/GetErrorMode):

Flag Value Description
SEM_FAILCRITICALERRORS 0x0001 System does not display critical-error-handler message box

🔗 Forwarder Example (C):

// This DLL contains NO actual implementation
// All error handling functions are forwarded to kernel32.dll

// Last-error management
#pragma comment(linker, "/export:GetLastError=kernel32.GetLastError")
#pragma comment(linker, "/export:SetLastError=kernel32.SetLastError")

// Exception handling
#pragma comment(linker, "/export:RaiseException=kernel32.RaiseException")
#pragma comment(linker, "/export:SetUnhandledExceptionFilter=kernel32.SetUnhandledExceptionFilter")
#pragma comment(linker, "/export:UnhandledExceptionFilter=kernel32.UnhandledExceptionFilter")

// Error mode control
#pragma comment(linker, "/export:GetErrorMode=kernel32.GetErrorMode")
#pragma comment(linker, "/export:SetErrorMode=kernel32.SetErrorMode")

⚠️ WARNING:

Error handling affects application behavior!
• SetErrorMode(SEM_NOGPFAULTERRORBOX) disables Windows Error Reporting — used by browsers and games
• SetUnhandledExceptionFilter is per-process — only one filter can be active
• RaiseException with EXCEPTION_BREAKPOINT (0x80000003) triggers debugger breakpoints
• GetLastError returns thread-local error codes — safe for multi-threading
• Error codes from GetLastError are documented in WinError.h (but many are undocumented!)
• For ReactOS: implement thread-local storage (TLS) for GetLastError correctness
