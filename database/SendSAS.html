SendSAS

UNDOCUMENTED / SYSTEM CRITICAL WIN 7 / 10 / 11 sas.dll

Description: This function programmatically triggers the Secure Attention Sequence (SAS), commonly known as the Ctrl+Alt+Del combination. It is a fundamental Windows security element that ensures a transition to the Secure Desktop. It is used by system utilities, screen readers, and remote desktop software to communicate directly with Winlogon via protected channels.

C# P/Invoke Structure:

[DllImport("sas.dll", SetLastError = true, CharSet = CharSet.Unicode)]
public static extern void SendSAS(
[MarshalAs(UnmanagedType.Bool)] bool AsUser
);

Possible NTSTATUS Codes:

0x00000000 - STATUS_SUCCESS - The SAS signal was successfully sent via WmsgSendMessage (Message ID: 520). [0x00000000]
0xC0000022 - STATUS_ACCESS_DENIED - Caller lacks SeTcbPrivilege or the executable manifest is missing uiAccess="true". [0xC0000022]
0xC0020006 - RPC_NT_UNKNOWN_IF - The SAS RPC interface (52EF130C-...) was not found. Winlogon may be unresponsive. [0xC0020006]
0xC0000041 - STATUS_PORT_DISCONNECTED - The ALPC connection to the Winlogon ApiPort was closed unexpectedly. [0xC0000041]
0xC0000602 - STATUS_INTERNAL_ERROR - An internal logic failure occurred while allocating the RPC binding handle. [0xC0000602]

⚠️ WARNING:

For a successful call, the application MUST be digitally signed and contain a manifest with uiAccess="true". Otherwise, the system will block the call to prevent phishing. User-mode calls are redirected via RPC to the 52EF130C-08FD-4388-86B3-6EDF interface, which may introduce latency.
