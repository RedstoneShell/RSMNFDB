NtVdm64CreateProcessInternalW

UNDOCUMENTED WIN 7 / NT 6.1 ntvdm64.dll

Description: Core function that launches 16-bit executables (NE format) on 64-bit Windows through NTVDM. Performs AppCompat checks, resolves the full path, queries the registry for application-specific settings (CommandLine, ProductName, InternalName, ProductVersion, MappedExeName), expands environment variables, updates the __PROCESS_HISTORY tracking variable, and finally calls CreateProcessInternalW in kernel32.dll to spawn the NTVDM process. This is the primary entry point for all 16-bit process creation on x64 systems.

C# P/Invoke Structure:

[DllImport("ntvdm64.dll", SetLastError = true, CharSet = CharSet.Unicode)]
public static extern int NtVdm64CreateProcessInternalW(
    IntPtr hProcess,                // Process handle (usually NULL)
    string lpCommandLine,           // Command line to execute
    IntPtr lpProcessAttributes,     // Process security attributes
    IntPtr lpThreadAttributes,      // Thread security attributes
    bool bInheritHandles,           // Inherit handles flag
    uint dwCreationFlags,           // Creation flags
    IntPtr lpEnvironment,           // Environment block
    string lpCurrentDirectory,      // Current directory
    IntPtr lpStartupInfo,           // STARTUPINFO structure
    IntPtr lpProcessInformation,    // PROCESS_INFORMATION structure
    IntPtr lpUnknown1,              // Internal VDM parameters
    IntPtr lpUnknown2               // Internal VDM context
);

⚠️ WARNING:

This function is only available on 32-bit Windows and 64-bit Windows with NTVDM enabled (deprecated since Windows 10). On 64-bit Windows without NTVDM, it will display an error message box and fail. Requires SeDebugPrivilege for debugging 16-bit processes. Modifying the registry keys under HKLM\Software\Microsoft\Windows NT\CurrentVersion\NtVdm64 can cause system instability.
