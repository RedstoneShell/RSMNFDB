api-ms-win-crt-multibyte-l1-1-0.dll

API SET FORWARDER UCRTBASE.DLL Windows 7 / NT 6.1+ 199 exports

Description: This is an API Set forwarder DLL for multibyte character set (MBCS) handling in the Universal C Runtime. It contains NO executable code — only export forwarders to ucrtbase.dll. These functions provide support for double-byte character sets (DBCS) used in Asian languages (Japanese, Chinese, Korean) and MBCS string manipulation functions.

Why it exists: This API Set provides the complete MBCS string library from the C Runtime. It includes functions for character classification, conversion between single-byte and double-byte characters, Japanese-specific conversions (Hiragana/Katakana, JIS/Shift-JIS), and safe string manipulation functions. Essential for applications that need to support Asian language environments.

⚠️ WARNING:

MBCS functions are locale-dependent!
• These functions behave differently based on the current locale (Japanese, Chinese, Korean, etc.)
• The lead byte tables (__p__mbctype) define which bytes are valid lead bytes in DBCS
• Japanese-specific conversions (_mbctohira, _mbcjistojms) only work in Japanese locale
• Many of these functions are deprecated in favor of Unicode/Wide character equivalents
• For modern applications, use Unicode (UTF-16) or UTF-8 instead of MBCS
• On Windows 7, Universal C Runtime requires KB2999226 or Visual Studio 2015 redistributable
• For ReactOS: implement MBCS tables for Asian locales to support legacy applications
