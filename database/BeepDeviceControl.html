BeepDeviceControl

UNDOCUMENTED WIN 7 beep.sys

Description:Function BeepDeviceControl is a specially DeviceIOControl invokable function, to beep a PC Speaker, can not work :(.

C++ Raw Structure:

NTSTATUS BeepDeviceControl(
    PDEVICE_OBJECT DeviceObject,
    PIRP Irp
)

C# P/Invoke Structure:

NTSTATUS BeepDeviceControl(
    PDEVICE_OBJECT DeviceObject,
    PIRP Irp
)

Associated Structure:

IntPtr h = CreateFile("\\\\.\\Beep", ...);

BEEP_SET_PARAMETERS beep = new BEEP_SET_PARAMETERS
{
    Frequency = 1000,
    Duration = 500
};

DeviceIoControl(
    h,
    0x00010000, // Beep command, in Windows other versions this code can be not equals this
    ref beep,
    Marshal.SizeOf(beep),
    IntPtr.Zero,
    0,
    out _,
    IntPtr.Zero
);

Possible NTSTATUS Codes:

0x00000000 - STATUS_SUCCESS - The driver successfully accepted the parameters and initiated the beep.
[0x00000000]
0xC000000D - STATUS_INVALID_PARAMETER - Frequency is out of range (37Hz - 32767Hz) or the data structure is malformed.
[0xC000000D]
0xC0000022 - STATUS_ACCESS_DENIED - Insufficient privileges. The caller must have administrative rights or SeDebugPrivilege.
[0xC0000022]
0xC0000004 - STATUS_INFO_LENGTH_MISMATCH - The input buffer size does not match the size of BEEP_SET_PARAMETERS (8 bytes).
[0xC0000004]
0xC0000010 - STATUS_INVALID_DEVICE_REQUEST - The IOCTL code provided is not 0x10000 or is unsupported by this device object.
[0xC0000010]
0xC000009A - STATUS_INSUFFICIENT_RESOURCES - The kernel failed to allocate a timer or non-paged pool memory to process the beep.
[0xC000009A]
0xC0000120 - STATUS_CANCELLED - The beep request was terminated before completion (e.g., a new beep command overrode it).
[0xC0000120]

⚠️ WARNING:

Run as Admin: Without this, CreateFile return INVALID_HANDLE_VALUE (Err 5: Access Denied). If "Frequency" parameter in structure <37 or >32767, driver return STATUS_INVALID_PARAMETER
