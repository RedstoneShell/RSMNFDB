CheckSnapshotPresence

UNDOCUMENTED WIN 7 / NT 6.1 ifsutilx.dll

Description: The CheckSnapshotPresence function verifies the current status and existence of a previously created volume shadow copy. It acts as a lightweight validation routine by querying the volsnap.sys driver via the internal IVssBackupComponents::GetSnapshotProperties method. This is essential for long-running operations to ensure the snapshot has not been automatically deleted by the system (e.g., due to storage limits or provider timeouts).

C# P/Invoke Structure:

      [DllImport("ifsutilx.dll", CallingConvention = CallingConvention.FastCall)]
      public static extern bool CheckSnapshotPresence(IntPtr snapshotData);

Parameters:

  • snapshotData (IntPtr): A pointer to the _SNAPSHOT_DATA structure returned by CreateVolumeSnapshot.

Return Value:

Returns true if the shadow copy device is still registered and active in the Object Manager. Returns false if the snapshot was deleted, the pointer is corrupt, or the VSS session has timed out.

Possible NTSTATUS Codes:

0x00000000 - STATUS_SUCCESS - The snapshot is present and the metadata matches the current system state.[0x00000000]
0xC0000005 - STATUS_ACCESS_VIOLATION - Invalid pointer provided. The memory block at snapshotData could not be read.[0xC0000005]
0xC0040001 - STATUS_VSS_VOLSNAP_NOT_FOUND - The specific snapshot ID was not found in the volsnap.sys database (Snapshot was likely deleted).[0xC0040001]
0xC000000D - STATUS_INVALID_PARAMETER - The Magic number in the structure is not 0x1, indicating an invalid or uninitialized SNAPSHOT_DATA block.[0xC000000D]

⚠️ WARNING:

Performance Impact: Avoid calling this function in a tight loop. Each check involves an IOCTL call to the kernel and potentially an RPC call to the VSS service, which can introduce latency. False Positives: A true result only guarantees that the snapshot existed at the moment of the call. In high-load scenarios, the snapshot could be evicted immediately after the function returns. Data Integrity: This function does not check the integrity of the data within the snapshot; it only verifies that the volume device object (e.g., \Device\HarddiskVolumeShadowCopyX) is still registered in the object manager.
